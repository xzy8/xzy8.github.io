JavaScript笔记

1. JavaScript概述

1.1 JavaScript 是什么

  • JavaScript 是世界上最流行的语言之一，是一种运行在客户端的脚本语言 （Script 是脚本的意思）
  • 脚本语言：不需要编译，运行过程中由 js 解释器( js 引擎）逐行来进行解释并执行
  • 现在也可以基于 Node.js 技术进行服务器端编程

1.2 JavaScript 的作用

  • 表单动态校验（密码强度检测） （ JS 产生最初的目的 ）
  • 网页特效
  • 服务端开发(Node.js)
  • 桌面程序(Electron)
  • App(Cordova)
  • 控制硬件-物联网(Ruff)
  • 游戏开发(cocos2d-js)

1.3 JavaScript的组成

  • ECMAScript

    ECMAScript 是由ECMA 国际（ 原欧洲计算机制造商协会）进行标准化的一门编程语言，这种语言在万维网上应用广泛，它往往被称为 JavaScript 或 JScript，但实际上后两者是 ECMAScript 语言的实现和扩展。

  • DOM ——文档对象模型

    文档对象模型（Document Object Model，简称DOM），是W3C组织推荐的处理可扩展标记语言的标准编程接口。通过 DOM 提供的接口可以对页面上的各种元素进行操作（大小、位置、颜色等）。

  • BOM ——浏览器对象模型

    BOM (Browser Object Model，简称BOM) 是指浏览器对象模型，它提供了独立于内容的、可以与浏览器窗口进行互动的对象结构。通过BOM可以操作浏览器窗口，比如弹出框、控制浏览器跳转、获取分辨率等。

1.4 JS的3种书写位置

  • 行内式 JS

    • 可读性差， 在html中编写JS大量代码时，不方便阅读
    • 引号易错，引号多层嵌套匹配时，非常容易弄混
    • 特殊情况下使用

  • 内嵌 JS

    • 可以将多行JS代码写到 script 标签中
    • 内嵌 JS 是学习时常用的方式

  • 外部 JS文件

    • 利于HTML页面代码结构化，把大段 JS代码独立到 HTML 页面之外，既美观，也方便文件级别的复用
    • 引用外部 JS文件的 script 标签中间不可以写代码
    • 适合于JS 代码量比较大的情况

      • 如果数字后面有其它字符也会被省略
      • 如果变量不是数字开头转换后会变成NaN

      • parseFloat(变量名,进制数)可以通过进制转换，进制数为10,8,2等

      • null值转换后是0
      • undefined转换后是NaN
      • 空串和全是全是空格转换后是0

    • 隐式转换通过 * - \运算

      • *任何值做 - \操作时都会先转换为数字型**

    • 两种强制转换的去比特

3. 运算符

3.1 算术运算符

  • 运算符+
    • 任何值和字符串做 “+”运算都是拼接运算**(与Java中类似)**
    • 当两个小数做”+”运算时会有误差，譬如0.1+0.2不等于0.3,其它四种运算也会产生误差**(Java中double类似)**
    • 没有字符串参与则会将值转换为number类型再做”+”运算
    • 任何值和NaN运算结果都是NaN
  • 运算符 - * / %
    • 纯数值型的字符串可以做以上四种操作
    • 如果字符串不是纯数值型的则运算结果是NaN
  • 前置自增/自减运算
    • 先自增/自减，再运算
  • 后置自增/自减
    • 先运算，再自增/自减
  • 纯数值不能自增或/自减
    • 比如2++是错误写法

3.2 比较运算符

  • 两个变量比较时会默认转换为number型

    • 如果两个值都是字符串型，按照数据字典逐位比较(比如’112’<’12’)

  • 任何值与NaN比较都是false，包括NaN自身

  • null转换不明确

    • 比如1 > null为true
    • -1>null为false
    • 但是0 == null为false

  • 其他特殊的情况

  • ==/!= 和 ===/!==的区别

    • 前者是忽略数据类型，只要数值相等就返回true
    • 后者不忽略数据类型，必须类型与数值都相等才返回true

3.3 逻辑运算符

  • 如果是两个非布尔值的变量运算则会隐式转换为布尔类型

  • &&逻辑与运算

    • 所有值都为true才为true
    • 有短路机制，第一个表达式如果为false则不会执行后面表达式
    • JS中逻辑与的特殊机制
      • 当结果是true时返回最后一个为true的表达式的值
      • 当结果是false时返回第一个为false的表达式的值

  • ||逻辑或运算

    • 所有都为false才为false

  • 有短路机制，第一个表达式如果为true则不会执行后面表达式

    • JS中逻辑或的特殊机制

  • 当结果是false时返回最后一个为false的表达式的值

    • 当结果是true时返回第一个为true的表达式的值

4. 4 switch-case 和 if-else 的区别

  • 一般情况下，它们两个语句可以相互替换

  • switch…case 语句通常处理 case为比较确定值的情况， 而 if…else…语句更加灵活，常用于范围判断(大于、等于某个范围)

  • switch 语句进行条件判断后直接执行到程序的条件语句，效率更高。而if…else 语句有几种条件，就得判断多少次。

  • 当分支比较少时，if… else语句的执行效率比 switch语句高。

  • 当分支比较多时，switch语句的执行效率比较高，而且结构更清晰

  • push(参数…)在尾部添加一个或多个元素，会返回添加后的数组长度
  • pop()删除数组尾部的元素,并将删除的元素返回

  • reverse()反转数组，对原数组造成影响
  • concat(数组1,数组2)，拼接几个数组变成一个数组，不会对原数组造成影响
  • join(separator)，将一个数组转成字符串，每个元素通过separator分隔，不会对原数组造成影响

9. 事件

9.1 事件流

  • 冒泡流

    • 事件响应函数从子元素开始执行，然后向父节点传播

  • 捕获流

    • 事件从最上层开始执行向下层子节点传播

9.2 三种绑定方式

  • html级别绑定

    • 在HTML中内嵌js程序
    • 不符合DOM标准，无法设定事件流，默认冒泡流
    • 代码耦合性高

  • DOM0级别绑定

    • 同一个事件只能绑定一个相应函数
    • 无法传递参数
    • 无法设置事件流，默认冒泡流
    • 通过obj.onXXXX = null解绑

  • DOM2级别绑定

    • 同一个事件可以绑定多个响应函数，按照绑定顺序执行
    • 第一个参数时string，表示事件类型，不用写on，第二个函数就是监听函数就是该事件的具体事件，第三个参数可以使用捕获流true，false和默认为冒泡流
    • 监听函数可以传一个evert参数，event调用stopPropagation()，该方法可以组织事件继续传播

9.3 几种常见的事件

  • 页面加载事件onload()

    • html页面是按照顺序执行
    • 调用window.onload 方法可以避免出现页面未加载就执行js语句的问题
    • 也可以将script标签写在body内容下方避免上述问题

  • 鼠标事件

    • onclick() 鼠标单击事件
    • ondblclick() 双击鼠标事件
    • onmouseover() 鼠标悬停事件
    • onmouseout() 鼠标移入事件
    • onmouseup() 鼠标抬起事件
    • onmousedown() 鼠标下落事件
    • onmousemove() 鼠标移动事件
    • onblur() / onfocusout() 失去焦点事件
    • onfocus()/ onfocusin() 获得焦点事件
  • 键盘事件
    • onkeyup() 键位抬起事件（一直按住不抬起不算触发）
    • onkeydown() 键位按下事件
  • 其它常用事件
    • onchange() 内容改变事件
    • onsubmint() 点击提交事件
    • setIntervel(‘方法名()’,时间间隔) 计时触发方法的事件

10.2 几种常用查询方式1

  • 查询注意：只有在页面加载后才能查询到

    • 在window.onload 方法中查询
    • 或者在body下面查询

  • 方法

    • getElementById() 通过id查询，返回的是一个对象
    • getElementsByTagName() 通过标签名查询,返回的是一个数组
    • getElementsByName() 通过name属性查询，返回的是一个数组
    • getElementsByClassName() 通过class属性查询，返回的是一个数组

  • 属性

    • value 获取单标签的value值(比如表单中的输入框等)
    • innerHTML 获取标签内的值(包括文本元素和标签元素)
    • innerText 获取标签内的文本信息(只有文本，没有其他标签)

    • childNodes 获取所有子节点，包括文本元素(空格等)

    • children 获取所有子标签元素，不包括文本元素

    • firstChild 获取第一个子节点，包括文本元素(空格等)

    • firstElementChild 获取第一个子标签元素，不包括文本元素

    • lastChild 获取最后一个子节点，包括文本元素(空格等)

    • lastElementChild 获取最后一个子标签元素，不包括文本元素

    • parentNode 获取父节点
    • previousSibling 获取前一个兄弟节点，包括文本元素(空格等)
    • previousElementSibling 获取前一个兄弟元素，不包括文本元素
    • nextSibling 获取后一个兄弟节点，包括文本元素(空格等)
    • nextElementSibling 获取后一个兄弟元素，不包括文本元素

10.2 几种常用查询2

  • 方法
    • querySelector() 通过选择器查询，如果有多个结果也只会返回第一个
    • queryAllSelector() 查询所有符合的选择器，即使只有一个也封装在数组里
  • 属性
    • all 返回页面所有标签元素
    • body 返回该文档的body对象
    • title 返回该文档的标题对象
    • documentElement 返回html对象
